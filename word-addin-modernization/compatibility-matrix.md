## Dependency And Call Chain Notes

- ETKPlus wrapper macros use `WordBasic.Call "ETKPlusLibrary.RegReminder"` as a registration/reminder gate. This should be dropped from the Office.js MVP.
- ETKPlus wrappers call `Common.CheckRevisions` and `Common.ResetRevisions`; those helpers store `TrackRevisions` and `ShowRevisions` via `SaveSetting`/`GetSetting` and mutate `ActiveDocument`. Office.js MVP should use explicit user warnings instead of hidden revision-state toggles.
- ETKPlus wrappers call `Common.CheckView`/`ResetView`; those helpers use `ActiveWindow.View`, `Application.ScreenUpdating`, and registry persistence. Office.js MVP should not change the Word view as a cleanup precondition.
- ETKPlus `ReplaceInStoryRange` depends on public `rngStory` populated by higher-level FileCleaner orchestration, then runs `rngStory.Find.Execute Replace:=wdReplaceAll`. A new adapter must own explicit range selection/search behavior instead of depending on global range state.
- ETKPlus candidate implementations use `Application.UndoRecord`. Office.js has no direct equivalent for custom undo grouping, so acceptance should rely on narrow selected-range operations, preview, and Word's normal undo stack.
- TransTools `Trans_DocCleaner` depends on `frmDocCleaner.frm`, modeless UserForms, and `GetActiveApplicationWindowHandle(Application)`. This is a UI redesign candidate, not a direct port.
- TransTools `DocCleaner.pCleanTags` orchestrates progress classes, selection restore, story ranges, shapes, headers/footers, bookmark deletion, font normalization, and optional revision acceptance. It is too broad for MVP.
- TransTools `modDocCleaner.bas` contains native Windows API declarations (`kernel32`, `shfolder`, `user32`) and uses additional native clipboard helpers indirectly. These are `VSTO-only` infrastructure if exact desktop parity is ever required.
- TransTools settings/profile behavior is mostly in `frmDocCleaner.frm` and other forms via `VBA.GetSetting`/`VBA.SaveSetting`, not in the two candidate `.bas` files alone. If UI/profile behavior is later redesigned, settings must move to add-in storage.
- TransTools shape and text-frame support is richer than the ETKPlus MVP candidates. Treat it as future fixture scope, not first MVP behavior.
